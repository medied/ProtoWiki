# Wiki loop ➰ 

As part of the #sprinthackular2026-retain-contributors week effort, I explored ways to loop users back into Wikipedia. The core idea is that editors return not only when they see data, but when they see themselves in a story that is still unfolding. So turning data signals into a user story that can be layered on top of Wikipedia, injecting more personal meaning into an existing page.

We can use different WikiSignals as a trigger for a narrative moment (or a "loop"). Whatever the user did in their last visit would trigger a new appropriate loop, a cause and effect dynamic, the loop would be "completed" if the user lands back on Wikipedia. 

This idea would lend itself for a test/experiment to measure if it actually loops users back in and drives engagement up. I haven't properly researched what has been tried/tested in this area at the Foundation before.
